**very classic problem
## 相关链接 ：
atcode : [E - Fill-Rect Query](https://atcoder.jp/contests/abc464/tasks/abc464_e)
## 题目大况：
**题目给定一个n,m的矩阵，初始的时候时，矩阵都时字符‘A’;

**现在进行q次操作，操作如下**：
1.给定一个矩阵去覆盖在这个n,m的矩阵中，覆盖的字符为给定的 C ,覆盖的范围为，给定这个矩阵的右下角，同时这个矩阵的左上角一定为（1，1）-> (based -> 1)  这个范围；
2.现在需要你完成，进行完q次操作时的最终结果；

## 思路：
**暴力思考：每次都去遍历需要覆盖的矩阵，将这个点改为我在这次操作时需要覆盖的字符，最后输出；（TIE）;**

**问题1：这里我思考这么优化？
思考这个一个节点最终的输出状态由什么决定，是不是这个节点上的所有操作中的最后一次的覆盖状态，也就是说这个节点从后往前当这个节点还没有覆盖时，此时的操作就是节点的最终状态，否则跳过，无需进行往后的任何操作；

**问题2：那我这么样去把这个操作，优化到（n * m）？
数据结构优化
用个set记录每一行中还没有满的行下标，同时lim数组`lim[i]`表示在行下表为i时的最后一个被覆盖的列位置；
当`lim[i]`覆盖到了最后也就是`lim[i] == m`时，此时set里面去掉i,表示下标为i的行已经填满，

**最后就可以做到（n * m）的时间复杂度，输出即可；

## 代码如下：
```c++
#include <bits/stdc++.h>
using namespace std;
#define int long long
const int mod = 1e9 + 7;
const int N = 3e5 + 7;

void solve()
{
    int n, m, q;
    cin >> n >> m >> q;
    vector<tuple<int, int, char>> op(q);
    for (int i = 0; i < q; i++)
    {
        int l, r;
        char c;
        cin >> l >> r >> c;
        op[i] = {l, r, c};
    }
    vector<vector<char>> g(n + 1, vector<char>(m + 1, 'A'));
    vector<int> lim(n + 1, 0);
    set<int> st;      
    for (int i = 1; i <= n; i++) st.insert(i);

    for (int k = q - 1; k >= 0; k--) {
        auto [l, r, c] = op[k];
        auto it = st.begin();
        while (it != st.end() && *it <= l) {
            int x = *it;
            it++;
            int start = lim[x] + 1;
            if (start > r) continue;
            for (int j = start; j <= r; j++) g[x][j] = c;
            lim[x] = r;
            if (lim[x] == m) st.erase(x);
        }
    }
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= m; j++) cout << g[i][j];
        cout << endl;
    }
    return;
}
signed main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(0);
    int T = 1;
    while (T--)
    {
        solve();
    }
    return 0;
}
```

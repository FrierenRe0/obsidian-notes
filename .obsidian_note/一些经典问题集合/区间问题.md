# 区间合并 --> 看区间的左端点

**解释：因为我想要把所有的区间都能看到，则在这里我思考时可以简单得出优先看左端点会比较好，所以在这里我们对左端点进行排序，排完序之后，保证了在后续遍历每个区间时，会把我所遍历的区间维护好（或者说不会存在左侧还有区间没覆盖到）；

**思路：
1.先建立初始区间
2.在遍历后续区间
- **（无重叠），没有重叠时，就把这个分段区间维护好，在更换新区间 
- **（重叠）， 有两种情况；
	-  完全重合 -> 不做处理（原因：该区间已经被记录了） 
	-  部分重合 -> 只更新右区间 （原因： 该区间已经经过了左端点排序，会存在左侧还有区间没覆盖到，只更新右端点，维护完整就行）；
	-

## 合并区间
```c++
class Solution {
public:
    int N = 1e8;
    vector<vector<int>> merge(vector<vector<int>>& intervals) {
        intervals.push_back({N,N});
        sort(intervals.begin(),intervals.end());
        int begin = intervals[0][0],end = intervals[0][1];
        vector<vector<int>> ans;
        for(int i = 1;i < intervals.size();i++) {
            if(end < intervals[i][0]) { 
                vector<int> cur(2);
                cur[0] = begin;
                cur[1] = end;
                ans.push_back(cur);
                begin = intervals[i][0];
                end = intervals[i][1];
            }
            else if(end < intervals[i][1]) end = intervals[i][1];
        }
        return ans;
    }
};
```

# 区间选择 --> 看区间的有端点

**参考cf中南9A题题解
思考方向：
要理解贪心和dp的本质区别：
贪心一定是从局部最优推至全局最优；
dp则不是如此，局部最优也许时推不到全局最优的 —> 而是存在我这个点不选可以换取后面的跟到收益问题 —>这里也就是引出了dp的标志：“选与不选” —> 在这里便会存在大问题到子问题的递归/或者采用状态转移方程迭代

### 无重叠区间
```c++
bool cmp(const vector<int>& a,const vector<int>& b) {
    if(a[1] != b[1]) {
        return a[1] < b[1];
    }
    else {
        return a[0] < b[0];
    }
}
class Solution {
public:
    int eraseOverlapIntervals(vector<vector<int>>& intervals) {
        sort(intervals.begin(),intervals.end(),cmp);
        int mx = 1;
        int end = intervals[0][1];
        for(int i = 1;i < intervals.size();i++) {
            if(end <= intervals[i][0]) {
                mx++
                end = intervals[i][1];
            }
        }
        return intervals.size() - mx;
    }
};
```

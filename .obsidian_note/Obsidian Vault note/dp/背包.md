# 01背包
本质上是 : 对于一个给定一个限制性条件，在这个限制性条件中每个元素具有选择性，在选择性中规划出极值答案
有下面常见的3中情况：
![配图](../../截图文件/屏幕截图%202026-04-07%20120732.png)
一般是关于求这3种问题：1.求最大   2.求最小   3.求方案数
这三种决定了他们不同的状态转移方程：(递推)

二维`dp`：
->这里个人更加偏向写二维`dp`数组
好处：
- 每一次状态递推的过程是相邻的层级关系，没有了优化后的状态压缩
坏处：
- 有内存开销更大

1.求`min/max`
```c++
if(j < nums[i]) dp[i][j] = dp[i - 1][j];//只能不选

// 但在求最小值时，若无法不选（必须装满），此状态应设为无穷大//非法状态

else dp[i][j] = max(dp[i - 1][j],dp[i - 1][j - nums[i]] + nums[i]);

 //dp[i][j] = min(dp[i-1][j], dp[i-1][j-nums[i]] + nums[i]);
注意：
当然不一定加nums[i]
求max值时，+= nums[i];
求max_len时，+= 1;//跟着题目来

```
**初始化陷阱：求最小值时，DP 数组通常需要初始化为一个极大值（如 `0x3f3f3f3f`(INF)），而不是 0，因为我们要找 “最小” 的代价。(max同理)**
也就是非法状态要极限化，使得每一次状态转移时非法状态都没选上**
2.求方案数
```c++
dp[0][0] = 1;
if(j < nums[i]) dp[i][j] = dp[i - 1][j];

else dp[i][j] = d[i - 1][j] + dp[i - 1][j - nums[i]];
```


**但是如果这里有内存溢出：二维优化成一维/改成滚动数组(mod 2)**

一维`dp`：
1.求`min/max`
```c++
for(int i = 0;i < n;i++) {

	for(int j = m;j >= nums[i];j--) {

		dp[j] = max(dp[j],dp[j - nums[i]] + nums[i]);

		//dp[i][j] = min(dp[i - 1][j],dp[i - 1][j - nums[i]] + nums[i]);
	}
}
```
2.求方案数：
```c++
dp[0] = 1; // 👈 这一句就是【唯一必须的初始化】
for (int i = 0; i < n; i++) {

	for (int j = m; j >= nums[i]; j--) {

		dp[j] += dp[j-nums[i]];
	}
}
```

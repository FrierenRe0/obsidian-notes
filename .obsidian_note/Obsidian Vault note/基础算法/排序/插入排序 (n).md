# 插入排序完整总结

## 1. 时间复杂度
- **最好情况（数组已有序）**：\(O(n)\)
    每个元素只需和前一个对比一次，不用后移元素，单层循环。
- **平均  最坏情况（逆序）**：\(O(n^2)\)
    外层遍历 n 轮，内层每次最多移动全部前面元素。

## 2. 空间复杂度
\(O(1)\)，原地排序，仅常数临时变量。

## 3. 稳定性
**稳定排序**，相等元素相对顺序不变。

## 4. 核心思想
把数组分为「已排序前缀」和「未排序后缀」；
依次取出未排序第一个元素，向前遍历有序区找到插入位置，中间元素全部右移腾出空位，放入当前元素。
当然这里可以用链表避免中间数据整体后移带来的时间复杂度（例题:[对链表进行插入排序](https://leetcode.cn/problems/insertion-sort-list/)）


这里我想记录一下一种对于插入排序的特殊处理-->使用于当数据种类很小的时候
[颜色分类](https://leetcode.cn/problems/sort-colors/)
```c++
class Solution {
public:
    void sortColors(vector<int>& nums) {
        int p0 = 0,p1 = 0;
        for(int i = 0;i < nums.size();i++) {
            int x = nums[i];
            nums[i] = 2;
            if(x <= 1) {
                nums[p1++] = 1;
            }
            if(x == 0) {
                nums[p0++] = 0;
            }
        }
    }
};
//这套思路本质还是插入排序，只是做了极致优化，把普通数组插入排序「元素整体后移」的 \(O(n)\) 挪动操作，换成计数覆盖，省去大量元素交换，属于插入排序的改良版本。
```

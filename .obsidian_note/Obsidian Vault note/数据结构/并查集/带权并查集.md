**提问1：我想知道我在什么情况下我会想到带权并查集从而区别于普通并查集?**

普通并查集只记录连通关系，**带权并查集额外维护每个点到根节点的偏移量 `such as: dis[x]`**，用来存储 `x ~ root` 的差值，适用于等式约束类题目：比如变量差值、食物链、分数排名、等式方程等。
# 一、普通并查集能干什么？

只维护**连通关系**，只回答两个问题：

1. 两点是否在同一集合；
2. 合并两个集合。

    **没有任何数值、差值、权重信息**，只能判断 “连不连”。

适用场景：无任何大小 / 差值约束，只分块。

例：岛屿数量、朋友圈、连通块统计。

# 二、带权并查集（你这份模板）什么时候必须用？

只要题目存在**变量之间的差值 / 相对大小关系**，就不能用普通并查集，必须上带权并查集。

核心特征：给出形如 \(a - b = k\) 的等式约束。

## 典型场景分类

### 场景 1：等式差分约束（最常见）

题目给一堆 `x - y = c`，问是否存在矛盾。

例：

- 方程：`a - b = 3，b - c = 2，a - c = 4`，判断冲突；
- 分数：A 比 B 高 5 分，B 比 C 低 2 分，求 A 和 C 分差。

普通并查集只能知道 A、B、C 连通，但算不出分差，也校验不出矛盾。

### 场景 2：食物链 / 种类关系（模意义下带权）

经典食物链：3 类生物，A 吃 B，B 吃 C，C 吃 A。

本质是 `(a - b) mod 3 = k`，带权并查集维护模 3 偏移。

普通并查集完全做不了，无法区分捕食 / 同类 / 被捕食。

### 场景 3：区间前缀和、偏移关系

设 \(`s[i]`\) 是前缀和，条件 `s[r] - s[l-1] = val`，转换成 `s[r] - s[l-1] = val`，带权并查集维护前缀差。

### 场景 4：相对排名、坐标偏移

坐标：点 A 坐标比点 B 大 7；

排名：A 排名比 B 靠前 3 名。

都需要存储两点相对差值。

# 三、一眼区分：什么时候用普通 / 带权

## 只用普通并查集（无相对数值）

只关心连通与否，没有任何 “差多少、大多少、谁吃谁”：

1. 无权重、无数值约束；
2. 问题只问：是否连通、连通块个数、分组。

## 必须用带权并查集（存在相对关系）

题干出现下面任意一种描述：

1. x 比 y 大 / 小 k；
2. \(x - y = k\)；
3. x 和 y 相差 k；
4. 捕食、同类、敌对（模偏移）；
5. 区间和等于某个数（前缀差约束）；
6. 需要求出两点之间的差值。

# 四、核心底层区别

1. **普通并查集**

    数组：`fa[]`

    含义：父节点，无额外信息。

    find：只路径压缩，不更新任何附加数值。

    merge：只修改父节点，无计算。

2. **带权并查集（你的模板）**

    数组：`fa[] + dis[]`

    `dis[x]`：x 到根的差值 \(x-root\)

    find：递归路径压缩时**累加更新 dis**，保证每次查询 dis 是到根总偏移；

    merge：需要解方程算出两个根之间的偏移，给 `dis[根]` 赋值；

    同集合时要校验约束是否冲突。

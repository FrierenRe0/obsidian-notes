[GCD Graph](https://ac.nowcoder.com/acm/contest/133877/G)
### 题目：假设有一个图，节点是整数。从节点 $i$ 到节点 $j$ 的直接边权为 $\gcd(i, j)$。题目要求计算从区间 $[l, r]$ 内的每一个点 $i$，到达目标点 $n$ 的**最短路径长度之和**

**前置知识：** 埃式筛 + 组合数学 (容斥原理) + 求质因数 + dp

思路：
在 `[l,r]` 区间中每个节点存在如下情况：
- 如果一个数和n互质，说明该点走 gcd(i,n) = 1就一定为最短距离；

- 否则，如果不互质，说明该节点到n节点的直线距离一定gcd(i,n) > 1，所以如果存在一个节点u，gcd(i,u) = gcd(u,n) = 1，那就是该节点的最短路情况

- 最后一种情况，如果既不互质，又找不到”桥梁点 u”，就要想办法处理（但是对于数字敏感的人，这里会想到 dp，因为如果存在这种节点最多距离 n 节点的长度不会 > 30（在题目 1e7 的范围内），所以在 `[n-30, n-1]` 的这些节点可以使用 dp 的想法找到最短路径）

### 关键本题最巧妙的点就在于：可以区间分块 --> 为什么？
#### 一、 远处（距离 $n$ 超过 30 的点）：数论的“降维打击”

当一个点 $i$ 离 $n$ 很远时（即 $i < n - 30$），它想要走到 $n$，在图上有无数条路径。但数论定理告诉我们，**它的最短路径长度只有两种可能：要么是 1，要么是 2。** 绝对不可能出现 3 或更大的值。

我们来证明为什么：

1. **可能为 1 吗？**
    
    如果 $\gcd(i, n) = 1$（即 $i$ 和 $n$ 互质），那它们之间有一条直连边，边权就是 $\gcd(i, n) = 1$。那最短路当然就是 **1**。
    
2. **如果 $\gcd(i, n) > 1$（不互质），为什么一定是 2 呢？**
    
    如果不互质，直连边权就大于 1 了（比如可能是几千甚至几万）。但我们可以在 $[i+1, n-1]$ 中间找一个**中转点 $x$**。
    
    只要这个中转点 $x$ **同时与 $i$ 互质，且与 $n$ 互质**（即 $\gcd(i, x) = 1$ 且 $\gcd(x, n) = 1$）。
    
    那么路径 $i \to x \to n$ 的总代价就是 $1 + 1 =$ **2**。


**关键点来了：在远离 $n$ 的地方，这样的中转点 $x$ 极度好找！**

因为距离很远，中间有成百上千个连续整数。根据素数分布和互质的概率，你随便在中间捞几个数，就能轻松找到一个同时与 $i$ 和 $n$ 互质的“桥梁”。所以，远端不互质的点，其最短路必然能通过某个中转点缩短到 **2**。

既然远端点只有 `1` 和 `2` 两种答案，我们根本不需要大费周章地去跑图论算法或 DP！直接用前面讲的 `f` 函数（容斥原理）批量计算：

- 互质的数有几个（距离为 1）
    
- 不互质的数有几个（距离为 2）
    
- 这一大片区间直接 $O(1)$ 批量收割答案：`ts += (cc * 1 + ncc * 2);`


#### 二、 近处（距离 $n$ 在 30 以内的点）：数论规律失效，DP 接管

既然远端那么完美，为什么近处（靠近 $n$ 的地方）不能直接用这个结论呢？

**因为中间的“桥梁”不够用了！**

假设 $n = 10,000,000$，我们来看点 $i = 9,999,998$。

它离 $n$ 非常近。它们不互质（$gcd=2$）。如果我们想在它们中间找一个能把距离缩短到 2 的中转点 $x$，这个 $x$ 只能在它们狭窄的夹缝里挑。但因为夹缝太小了，可能根本找不到一个同时与 $i$ 和 $n$ 互质的数！

这时候，这个近处的点 $i$ 想要走到 $n$，它的最短路可能受到局部各种复杂的 $\gcd$ 关系的干扰，答案可能是 2、3、4 甚至更大，远端的“非 1 即 2”的定理在这里**失效了**。

既然数量很少（只有 30 个点），且规律复杂，代码就选择**强行用动态规划（DP）把这 30 个点的精准最短路算出来**


### 在距离n为30上为什么会存在一个划分点？--> 极端假设

#### 第一步：看看最极端的恶劣情况长什么样？

我们想要阻碍“桥梁点 $x$”的诞生，就得让 $i$ 和 $n$ 尽可能多地把数轴上的质数占满。

在 $10^7$ 以内：

- 一个数最多有 **8 个** 不同的质因子。
    
- 假设 $i$ 和 $n$ 极其倒霉，它们的质因子**完全没有重合**。
    
- 并且，它们把世界上**最小的 16 个质数**死死地全部占满了！


我们把这 16 个被“污染”的质数罗列出来：

> **$i$ 占领的质数（假设 8 个）：** $\{2, 5, 11, 17, 23, 31, 41, 47\}$
> 
> **$n$ 占领的质数（假设 8 个）：** $\{3, 7, 13, 19, 29, 37, 43, 53\}$

这就意味着，只要我们在中间挑的候选点 $x$ 能被这 16 个质数中的**任何一个**整除，这个 $x$ 就废了（不能当桥梁）。

#### 第二步：在长度为 30 的区间里“抓内鬼”

现在，我们在 $i$ 和 $n$ 之间切出一段**长度为 30 的连续整数区间**。我们来看看这 16 个恶鬼在这个区间里到底能破坏掉多少个数。

我们来做一次精细的算账：

1. **质数 2**：每 2 个数就破坏 1 个。在 30 个数里，它最多破坏 **15 个**。
    
2. **质数 3**：每 3 个数就破坏 1 个。在剩下的 15 个奇数里，它最多破坏 **5 个**。
    
3. **质数 5**：每 5 个数就破坏 1 个。在剩下的数里，它最多破坏 **2 个**。
    
4. **质数 7**：每 7 个数就破坏 1 个。在剩下的数里，它最多破坏 **1 个**。
    

我们把这几个最厉害的小质数干掉的格子加起来：$15 + 5 + 2 + 1 = 23$ 个。

30 个格子，被这 4 个小质数踩扁了 23 个，**还剩下 $30 - 23 = 7$ 个格子完好无损！**

**所以在最差的情况下，在距离 30 以外的节点都可以在距离 30 以内找到”桥梁点 u”，则在 30 距离以外 `[l,r]` 存在节点，则满足”互质为 1 非质为 2”的特点，所以在题目中，我可以把它定位为距离 n 为 30 的位置；**

## 代码思路的实现：
### 一  现在思路差不多了，处理左区间 

这里我们就要知道在这个区间中，存在有多少个数与 n 不互质（即至少与 n 共享一个质因子），这我们只需要看n对对应的质因子即可；
#### 1. 为什么“不互质”只需要看质因子？

先要明白互质的本质 ：说明对于这两个数的质因数集合的交集为空集；
否则，两者的交集存在实数；

反过来讲：**只要 $x$ 包含了 $n$ 的某一个“质因子”， $x$ 就注定与 $n$ 不互质。**

因此，我们在找 $[1, m]$ 中哪些数与 $n$ 不互质时，根本不需要考虑 $n$ （如n = 30）的合数因子（比如 6, 10, 15），**只需要死死盯住 $n$ 的那几个不同的质因子（2, 3, 5）就足够了**。


想要知道其因子是质数有要埃式筛预处理 + 再计算n的质数种类

埃式筛代码（模板） ：
```c++
const int N = 10000007;
int vis[N];
int prime[N/10],cnt;

void get_prime(int n) {
    vis[0] = vis[1] = 1;
    for(int i = 2;i < N;i++) {
        if(!vis[i]) {
            prime[++cnt] = i;
        }
        for(int j = 1;j <= cnt && prime[j] * i <= n;j++) {
            vis[prime[j] * i] = 1;
            if(i % prime[j] == 0) break;
        }
    }
}
```

计算n的质数种类（模板）：
```c++
int factor[10],num; //在范围1 ~ 1e7中质因子的个数最多就是8

void get_factor(int n) {
    num = 0;
    int temp = n;
    for(int i = 1;prime[i] * prime[i] <= temp && i <= cnt;i++) {
        if(temp % prime[i] == 0) {
            factor[num++] = prime[i];
            while(temp % prime[i] == 0) temp /= prime[i];
        }
    }
    if(temp > 1) factor[num++] = temp;
}
```

计算不互质的元素个数这里就需要用到一些数学原理 （容斥原理）+ 状态压缩
```c++
int f(int m,int num) { 
    //容斥原理 + 组合数学 
    //解决问题：计算 1 到 m 中，有多少个数与 n 不互质
    if(m <= 0) return 0;
    int res = 0;
    for(int i = 1;i < (1LL << num);i++) {
        int sum = 0,temp = 1;
        for(int j = 0;j < num;j++) {
            if((i >> j) & 1) {
                sum++;
                temp *= factor[j]; //lca --> 计算选取质数的最小公倍数
            }
        }
        // m / temp 就是对应该状态的交集数量大小
        if(sum % 2) res += m / temp; 
        else res -= m / temp;
    }
    return res;
}
```

### 二  处理右区间 --> dp

##### 问题: 为什么不会有先跳到前面的点上再跳回n点 ，也就是说为什么这里我只要看右边就可以找到最优情况?

用一个具体的反例来拆解
假设终点 $n = 100$。现在轮到我们计算节点 $i = 96$。
按照你的设想，有没有可能存在这样一条更优的路径：

> **96** $\longrightarrow$ **95（前面的点）** $\longrightarrow$ **100（终点）**

我们来算一算这条路径的真实代价（边权）：

- 第一步（$96 \to 95$）：边权是 $\gcd(96, 95) = 1$（因为相邻两个整数必定互质）。
    
- 第二步（$95 \to 100$）：边权是 $\gcd(95, 100) = 5$。
    
- **总代价**：$1 + 5 = 6$。
    

这确实是一条通路。但是，如果我们不往前看，而是**向后看（看 96 后面的点）**，能找到更优的吗？

在 96 后面，有 97。

- 第一步（$96 \to 97$）：边权是 $\gcd(96, 97) = 1$（相邻互质）。
    
- 第二步（$97 \to 100$）：边权是 $\gcd(97, 100) = 1$（互质）。
    
- **总代价**：$1 + 1 = 2$。
    

你会发现，向后看找到的路径（代价为 2）远远优于向前看再回跳的路径（代价为 6）。

决定性的数论铁律：相邻必互质

为什么每次“向前看”都占不到便宜？因为在这张图里，两点之间的边权是 $\gcd$。

1. **要想路径短，必须凑出 $\gcd = 1$（互质）。**
    
2. 每一个节点 $i$，它和它**左右紧挨着的邻居**（$i-1$ 和 $i+1$）的 $\gcd$ 都绝对等于 1。
    
3. 既然向左跨一步（到 $i-1$）的代价是 1，向右跨一步（到 $i+1$）的代价也是 1。那么我们为了去往右侧的终点 $n$，**为什么不直接选择向右跨呢？**
    

如果你选择向左跨到 $i-1$，你不仅付出了 1 的代价，还让自己**距离终点 $n$ 更远了**，接下来还要花更多的代价折返回来。这在数学上是绝对不划算的。

终极消除疑虑：如果前面的点“骨骼清奇”呢？

你可能会问：“万一前面的某个点 $j$，它跟终点 $n$ 的关系特别好，直连代价就是 1 呢？”

比如：$i = 96$，$n = 100$。前面有个点 $j = 97$……等等，**97 本来就在 96 的后面（右边）啊！**

仔细想想：

- 那些和终点 $n$ 互质（直连代价为 1）的“优秀中转点”，是**均匀分布**在整个数轴上的。
    
- 在 `i` 的左边有，在 `i` 的右边（也就是 `i` 和 `n` 之间）**也一定有**。
    
- 既然右边本来就有现成的、同样优秀的优秀中转点，`i` 为什么还要特意往左走去绕远路呢？

#### 这里我在dp的时候从右往左就可以找到对于该点的最优情况
```c++
int B = 30;
int split = max(l,n - B);
    //将区间一分为二
    //
    //  ----------|-------------
    //  |         |            |fadwdawdaw
    //  l         split        r     
    //
if(split < n) {
    for(int i = n - 1;i >= split;i--) {
        int cur = gcd(i,n);
        for(int k = i + 1;k < n;k++) {
            if(cur <= 2) break;
            cur = min(cur,gcd(i,k) + dp_cost[n - k]);
        }
        dp_cost[n - i] = cur;
        if(i >= l && i <= r) ans += cur;
    }
}
```

## 最终代码：

```c++
#include <bits/stdc++.h>
using namespace std;
#define int long long
const int mod = 1e9 + 7;
const int N = 10000007;

int vis[N];
int prime[N/10],cnt;
int factor[10],num;
int dp_cost[505];

void get_prime(int n) {
    vis[0] = vis[1] = 1;
    for(int i = 2;i < N;i++) {
        if(!vis[i]) {
            prime[++cnt] = i;
        }
        for(int j = 1;j <= cnt && prime[j] * i <= n;j++) {
            vis[prime[j] * i] = 1;
            if(i % prime[j] == 0) break;
        }
    }
}

void get_factor(int n) {
    num = 0;
    int temp = n;
    for(int i = 1;prime[i] * prime[i] <= temp && i <= cnt;i++) {
        if(temp % prime[i] == 0) {
            factor[num++] = prime[i];
            while(temp % prime[i] == 0) temp /= prime[i];
        }
    }
    if(temp > 1) factor[num++] = temp;
}

int f(int m,int num) { 
    //容斥原理 + 组合数学 
    //解决问题：计算 1 到 m 中，有多少个数与 n 不互质（即至少与 n 共享一个质因子）。
    
    if(m <= 0) return 0;
    int res = 0;
    for(int i = 1;i < (1LL << num);i++) {
        int sum = 0,temp = 1;
        for(int j = 0;j < num;j++) {
            if((i >> j) & 1) {
                sum++;
                temp *= factor[j];
            }
        }
        if(sum % 2) res += m / temp;
        else res -= m / temp;
    }
    return res;
}



void solve() {
    int l,r,n;
    cin >> l >> r >> n;
    get_factor(n);
    
    int ans = 0;
    int B = 30;
    int split = max(l,n - B);
    //将区间一分为二
    //
    //  ----------|-------------
    //  |         |            |fadwdawdaw
    //  l         split        r     
    //
    if(split < n) {
        for(int i = n - 1;i >= split;i--) {
            int cur = gcd(i,n);
            for(int k = i + 1;k < n;k++) {
                if(cur <= 2) break;
                cur = min(cur,gcd(i,k) + dp_cost[n - k]);
            }
            dp_cost[n - i] = cur;
            if(i >= l && i <= r) ans += cur;
        }
    }
    int far_r = min(r,split - 1);
    if(l <= far_r) {
        int len = far_r - l + 1;
        int ncc = f(far_r,num) - f(l - 1,num);
        int cc = len - ncc;
        ans += (cc + ncc * 2);
    }
    cout << ans << endl;
    return;
}
signed main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    cout.tie(0);
    int T = 1;
    get_prime(N - 1);
    cin >> T;
    while (T--) {
        solve();
    }
    return 0;
}
```
